Order READ

/v1/account/:account_id/orders/:order_id.json

Valid request

Verb GET
Example
curl https://api.eatabit.io/v1/account/839df546-cff4-4b41-999e-6cdfa6fd2e78/orders/1.json \
-X GET \
-u 839df546-cff4-4b41-999e-6cdfa6fd2e78:M4FmfvvHCCb5LmcLHWRcQg
Response code 200
Response body
JSON: {
"object" : "order",
"id" : 1,
"state" : "queued",
"pickup_minutes" : 15,
"delivery_minutes" : 30,
"items" : "One large pepperoni pizza|One dozen hot wings",
"fulfillment" : "delivery",
"delivery_address" : "123 Fake St. Apt 1A, Charleston, SC, 29401",
"prepaid" : true,
"payment" : "credit",
"customer_name" : "Hank Jones",
"customer_phone" : "+15551234567",
"customer_note" : "1st order",
"payment_note" : "####-####-####-0000",
"status_url" : "https://eatabit.io/orders/status.json",
"created" : "2013-11-27T15:26:43Z"
}

Request route parameters

:account_id
Position: /v1/account/:account_id/orders/1.json
Description: The UUID of your eatabit.io account. This is the same UUID as is passed as USER in the request HTTP Basic auth
:order_id
Position: /v1/account/839df546-cff4-4b41-999e-6cdfa6fd2e78/orders/:order_id.json
Description: The id of the order that was passed by you during Order creation.

Response parameters

object
Type: String
Value: "order"
Description: The class of the returned object. Provided for convenience.
id
Type: Integer
Description: The eatabit.io id for the order object. It is used for order lookup once the record is persisted.
state
Type: String
Description: The the current state of the order.
Possible values:
  • queued:
    • API has successfully recieved the order. Order is ready for download.
  • downloaded:
    • Order has been received successfully by the printer.
  • printed:
    • Restaurant has successfully printed the order.
pickup_minutes
Type: Integer
Description: The restaurant's current pickup time in minutes.
delivery_minutes
Type: Integer
Description: The restaurant's current delivery time in minutes.
items
Type: String
Description: The value of the items attribute.
fulfillment
Type: String
Description: A string specifying whether the order is pickup or delivery.
delivery_address
Type: String
Description: A string specifying the delivery address of the customer.
prepaid
Type: Boolean
Description: A boolean specifying whether the order prepaid.
payment
Type: String
Description: A string specifying whether the order payment type is cash, credit or in-store. Pickup orders are set to 'in-store'
customer_name
Type: String
Description: A string specifying the name of the customer.
customer_phone
Type: String
Description: A string with the customer phone number in E.164 format.
customer_note
Type: String
Description: A URL encoded string with an optional customer note.
payment_note
Type: String
Description: A URL encoded string with an optional payment note. Transmission of any credit card data, apart from the last 4 card number digits, is strictly forbidden.
status_url
Type: String
Description: A URL encoded string with the :status_url value you provided. Eatabit will POST to this url with the following params:

  • account_id:
    • The id (UUID) of your account.
  • id:
    • The id (UUID) of the order.
  • state
    • The current state of the order.
    • Possible values:
      • queued: The order has been queued for download.
      • downloaded: The order has been downloaded.
      • printed: The order has been printed.
created
Type: Timestamp
Description: The UTC time that the record was created. Timestamps are in iso8601 format.
