Authentication

API endpoint
https://api.eatabit.io
SSL
All requests should be made over HTTPS.
Authentication
All requests are authenticated with HTTP BASIC AUTH. The username is your account id and the password is your key. You can supply a DEVELOPMENT or PRODUCTION key.

Valid request

Verb GET
Example
curl https://api.eatabit.io/v1/account/1.json \
-X GET \
-u 1:6s3AcHtv4Eja2ZkEhtPo4w
Response code 200
Response body
JSON: {
"object" : "account",
"name" : "eatabit.io",
"support_phone_number" : "+15551234567",
"created" : "2013-11-27T15:26:43Z"
}

Account READ

/v1/account/:account_id.json

Valid request

Verb GET
Example
curl https://api.eatabit.io/v1/account/1.json\
-X GET \
-u 1:6s3AcHtv4Eja2ZkEhtPo4w
Response code 200
Response body
JSON: {
"object" : "account",
"name" : "eatabit.io",
"support_phone_number" : "+15551234567",
"created" : "2013-11-27T15:26:43Z"
}

Request route parameters

:account_id
Position: /v1/account/:account_id.json
Description: The id of your eatabit.io account. This is the same integer as is passed as USER in the request HTTP BASIC AUTH.

Response parameters

object
Type: String
Value: "account"
Description: The class of the returned object. Provided for convenience.
name
Type: String
Description: Your account's name. This string will appear on printed receipts.
support_phone_number
Type: String
Description: Your account's support phone number in E.164 format.
created
Type: Timestamp
Description: The UTC time that the record was created. Timestamps are in iso8601 format.

Order CREATE

/v1/account/:account_id/restaurant/:restaurant_id/orders.json

Valid request

Verb POST
Example
curl https://api.eatabit.io/v1/account/1/restaurant/+15551234567/orders.json \
-X POST \
-u 1:6s3AcHtv4Eja2ZkEhtPo4w \
-H 'application/x-www-form-urlencoded'
-d 'items=One%20large%20pepperoni%20pizza%7COne%20dozen%20hot%20wings' \
-d 'fulfillment=Delivery' \
-d 'delivery_address=123%20Fake%20St.%20Apt%201A,%20Charleston,%20SC,%2029401' \
-d 'prepaid=false' \
-d 'payment=credit' \
-d 'customer_name=Hank%20Jones' \
-d 'customer_phone=+15551234567' \
-d 'customer_note=1st%20order' \
-d 'payment_note=%23%23%23%23-%23%23%23%23-%23%23%23%23-0000'
Response code 200
Response body
JSON: {
"object" : "order",
"id" : 1,
"pickup_minutes" : 15,
"delivery_minutes" : 30,
"items" : "One large pepperoni pizza|One dozen hot wings",
"fulfillment" : "Delivery",
"delivery_address" : "123 Fake St. Apt 1A, Charleston, SC, 29401",
"prepaid" : false,
"payment" : "credit",
"customer_name" : "Hank Jones",
"customer_phone" : "+15551234567",
"customer_note" : "1st order",
"payment_note" : "####-####-####-0000",
"created" : "2013-11-27T15:26:43Z"
}

Request route parameters

:account_id
Position: /v1/account/:account_id/restaurant/+15551234567/orders.json
Description: The id of your eatabit.io account. This is the same integer as is passed as USER in the request HTTP BASIC AUTH.
:restaurant_id
Position: /v1/account/1/restaurant/:restaurant_id/orders.json
Description: The restaurant premise phone number in E.164 format (including the leading '+').

Request parameters

items
Required: Yes
Type: String
Description: A url-encoded string containing the order items. If a pipe ('|') delimited list is supplied, an ordered list will be printed on the receipt.
fulfillment
Required: Yes
Type: String
Values: 'Pickup', 'Delivery'
Description: A string specifying whether the order is pickup or delivery.
delivery_address
Required: Yes, if :fulfillment equals 'Delivery'
Type: String
Description: A URL encoded string specifying the delivery address of the customer.
prepaid
Required: Yes
Type: Boolean
Values: true, false
Description: A boolean specifying whether the order prepaid.
payment
Required: Yes
Type: String
Values: cash, credit
Description: A string specifying whether the order payment type is cash or credit.
customer_name
Required: Yes
Type: String
Description: A URL-encoded string specifying the name of the customer.
customer_phone
Required: Yes
Type: String
Description: A string with the customer phone number in E.164 format.
customer_note
Required: No
Type: String
Max length: 25 characters
Description: A URL encoded string with an optional customer note.
payment_note
Required: No
Type: String
Max length: 25 characters
Description: A URL encoded string with an optional payment note. Transmission of any credit card data, apart from the last 4 card number digits, is strictly forbidden.

Response parameters

object
Type: String
Value: "order"
Description: The class of the returned object. Provided for object/action verification convenience.
id
Type: String
Description: The client's internal id for the order object. It is used for order lookup once the record is persisted.
pickup_minutes
Type: Integer
Description: The restaurant's current pickup time in minutes.
delivery_minutes
Type: Integer
Description: The restaurant's current delivery time in minutes.
items
Type: String
Description: The value of the items attribute.
fulfillment
Type: String
Description: A string specifying whether the order is pickup or delivery.
delivery_address
Type: String
Description: A string specifying the delivery address of the customer.
prepaid
Type: Boolean
Description: A boolean specifying whether the order prepaid.
payment
Type: String
Description: A string specifying whether the order payment type is cash or credit.
customer_name
Type: String
Description: A string specifying the name of the customer.
customer_phone
Type: String
Description: A string with the customer phone number in E.164 format.
customer_note
Type: String
Description: A URL encoded string with an optional customer note.
payment_note
Type: String
Description: A URL encoded string with an optional payment note. Transmission of any credit card data, apart from the last 4 card number digits, is strictly forbidden.
created
Type: Timestamp
Description: The UTC time that the record was created. Timestamps are in iso8601 format.
