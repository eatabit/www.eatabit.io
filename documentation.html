Authentication

API endpoint
https://api.eatabit.io
SSL
All requests should be made over HTTPS.
Authentication
All requests must contain a valid :auth_key in the form of an http parameter. Your account has a DEVELOPMENT and a PRODUCTION key which are available in your login.

Valid request

Verb GET
Request
curl https://api.eatabit.io/v1/accounts \
-d "auth_key=6s3AcHtv4Eja2ZkEhtPo4w" \
-X GET
Response code 200
Response body
JSON: {
"object" : "account",
"name" : "eatabit.com",
"created" : "2013-11-27T15:26:43Z"
}

Invalid request

Verb GET
Request
curl https://api.eatabit.io/v1/accounts \
-X GET
Response code 400
Response body
JSON: {
"error" : "auth_key is missing"
}

/v1/orders

CREATE Order

Valid request

Verb POST
Request
curl https://api.eatabit.io/v1/orders \
-d "auth_key=6s3AcHtv4Eja2ZkEhtPo4w" \
-d "restaurant_id=1" \
-X POST
Response code 200
Response body
JSON: {
"object" : "order",
"id" : "1",
"pickup_minutes" : 15,
"delivery_minutes" : 30,
"created" : "2013-11-27T15:26:43Z"
}

Request parameters

auth_key
Required: Yes
Type: String
resturant_id
Required: Yes
Type: Integer

Response parameters

object
Type: String
Value: "order"
Description: The class of the returned object. Provided for object/action verification convenience.
id
Type: Integer
Description: The internal ID asigned to the Order object. Can be used to refer to the object in GET requests.
