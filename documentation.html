Authentication

API endpoint
https://api.eatabit.io
SSL
All requests should be made over HTTPS.
Authentication
All requests are authenticated with HTTP Basic auth. The username is your account id and the password is your key. You can supply a DEVELOPMENT or PRODUCTION key.

Valid request

Verb GET
Request
curl https://api.eatabit.io/v1/accounts/1 \
-X GET \
-u 1:6s3AcHtv4Eja2ZkEhtPo4w
Response code 200
Response body
JSON: {
"object" : "account",
"name" : "eatabit.com",
"support_phone_number" : "+15551234567",
"created" : "2013-11-27T15:26:43Z"
}

/v1/restaurants/:id.json

READ Restaurant

Valid request

Verb GET
Request
curl https://api.eatabit.io/v1/restaurants/15551234567.json \
-X GET \
-u 1:6s3AcHtv4Eja2ZkEhtPo4w \
Response code 200
Response body
JSON: {
"object" : "restaurant",
"id" : "15551234567",
"name" : "Bill's Pizza",
"premise_phone_number" : "+15551234567",
"created" : "2013-11-27T15:26:43Z"
}

Request parameters

auth_key
Required: Yes
Type: String
Description: Your account's PRODUCTION or DEVELOPMENT key.
id
Required: Yes
Type: Integer
Description: The eatabit.io internal id of the restaurant object.

Response parameters

id
Type: String
Description: The eatabit.io internal id of the restaurant object.
name
Type: String
Description: The name of the restaurant.
premise_phone_number
Type: String
Description: The phone number of the restaurant in E.164 format.
created
Type: Timestamp
Description: The UTC time that the record was created. Timestamps are in iso8601 format.

Order: CREATE

Valid request

Verb POST
Request
curl https://api.eatabit.io/v1/accounts/6s3AcHtv4Eja2ZkEhtPo4w/orders.json \
-X POST \
-u 1:6s3AcHtv4Eja2ZkEhtPo4w \
-d '{ \
"restaurant": { \
"id": "+15551234567" \
}, \
"order": { \
"external_id": 9999, \
"items": "One large pepperoni pizza|One dozen hot wings", \
"fulfillment": "pickup", \
"prepaid": true, \
"payment": "credit" \
}'
}'
Response code 200
Response body
JSON: {
"object" : "order",
"internal_id" : 1,
"external_id" : 9999,
"pickup_minutes" : 15,
"delivery_minutes" : 30,
"items" : "One large pepperoni pizza|One dozen hot wings",
"fulfillment" : "pickup",
"prepaid" : true,
"payment" : "credit",
"created" : "2013-11-27T15:26:43Z"
}

Request parameters

resturant_id
Required: Yes
Type: String
Description: The restaurant_id is the premise phone number of the restaurant in E.164 format. The premise phone number is the main, published
external_id
Required: Yes
Type: Integer
Description: The client's internal ID for the order object. It can be used for order lookup once the record is persisted.
items
Required: Yes
Type: String
Description: A string containing the order items. If a pipe ('|') delimited list is supplied, an ordered list will be printed on the receipt.
fulfillment
Required: Yes
Type: String
Values: 'pickup', 'delivery'
Description: A string specifying whether the order is pickup or delivery.
prepaid
Required: Yes
Type: Boolean
Values: true, false
Description: A boolean specifying whether the order prepaid.
payment
Required: Yes
Type: String
Values: cash, credit
Description: A string specifying whether the order payment type is cash or credit.

Response parameters

object
Type: String
Value: "order"
Description: The class of the returned object. Provided for object/action verification convenience.
internal_id
Type: Integer
Description: The API's internal ID for the order object. It can be used for order lookup once the record is persisted.
external_id
Type: Integer
Description: The client's internal ID for the order object. It can be used for order lookup once the record is persisted.
pickup_minutes
Type: Integer
Description: The restaurant's current pickup time in minutes.
delivery_minutes
Type: Integer
Description: The restaurant's current delivery time in minutes.
items
Type: String
Description: The value of the items attribute.
fulfillment
Type: String
Description: A string specifying whether the order is pickup or delivery.
prepaid
Type: Boolean
Description: A boolean specifying whether the order prepaid.
payment
Type: String
Description: A string specifying whether the order payment type is cash or credit.
created
Type: Timestamp
Description: The UTC time that the record was created. Timestamps are in iso8601 format.
